## 6. 추천 후보

추천 후보는 성장 포트폴리오 기반 “성장기록집” 서비스다.

### 문제

학습, 프로젝트, 회고, 실패, 개선 기록이 여러 도구에 흩어져 있다. 사용자는 기록을 계속 남기지만, 시간이 지난 뒤 자신의 성장 과정을 눈으로 확인하거나 설명 가능한 결과물로 만들기 어렵다.

### 사용자

- 취준생
- 주니어 직장인
- 대학생/학습자
- 공통점: 자신의 성장을 눈으로 확인하고, 필요할 때 포트폴리오나 회고 자료로 꺼내고 싶은 사람

### 서비스 방향

- 사용자가 성장 기록을 작성하거나 모은다.
- 기록을 프로젝트, 학습, 실패, 개선, 결과 같은 축으로 정리한다.
- AI는 기록을 대신 꾸며내지 않고, 챕터 구성, 요약, 누락 질문을 제안하는 보조 정리자 역할만 한다.
- 사용자는 선택한 기록을 “성장기록집”으로 주문한다.
- Lv3에서는 성장기록집 주문 1건에 필요한 기록, 챕터, 메타데이터를 JSON으로 익스포트한다.

## 7. 가정

- 성장 기록은 반복적으로 쌓이는 콘텐츠이므로 콘텐츠 서비스의 본체가 될 수 있다.
- 성장기록집은 책으로 전환되는 이유가 비교적 자연스럽다.
- AI가 보조 정리자 역할을 맡으면 AI 활용 문항에서 “최종 판단은 사용자”라는 설명이 가능하다.
- 이 주제는 지원자의 문제 정의, 검증, 회고, AI 활용 역량을 면접에서 설명하기 좋다.

## 8. 미확정 질문

- 최종 서비스명을 무엇으로 둘 것인가?
- 성장 기록의 기본 단위를 프로젝트, 회고, 이벤트 중 무엇으로 둘 것인가?
- 책 주문 시 사용자가 직접 챕터를 구성할지, 추천 구성을 먼저 보여줄지 결정해야 한다.
- 더미 데이터는 개발자 성장 사례로 둘지, 더 넓은 학습자 성장 사례로 둘지 결정해야 한다.

## 9. 원인 가설과 검증 계획

| 가설 | 검증 방법 |
|---|---|
| 사용자는 성장 기록을 남기지만 여러 도구에 흩어져 다시 정리하기 어렵다. | 더미 사용자 여정을 GitHub, Notion, 메모, 회고 기록 기반으로 구성해 플로우를 검증한다. |
| 성장 기록은 책으로 만들었을 때 “소장 가능한 변화의 증거”가 된다. | 성장기록집 주문 화면에서 선택 기록, 챕터, 기간, 제목이 자연스럽게 연결되는지 확인한다. |
| AI는 생성자가 아니라 보조 정리자일 때 과제와 면접에서 더 설득력 있다. | AI 제안 결과를 사용자가 수정/선택하는 흐름으로 제한할 수 있는지 제품 명세 단계에서 검토한다. |

## 10. 영향 규모

- 사용자 영향: 흩어진 기록을 성장 과정으로 다시 보고, 자기 설명과 회고에 활용할 수 있다.
- 사업적 영향: 디지털 기록을 실물 성장기록집으로 전환하는 주문 흐름을 만들 수 있다.
- 과제 평가 영향: Lv1 콘텐츠 서비스, Lv2 주문 상태 관리, Lv3 주문 데이터 익스포트가 하나의 흐름으로 설명된다.

## 11. 범위

포함:

- 성장 기록 생성, 조회, 수정, 삭제 또는 이에 준하는 핵심 플로우
- 성장 기록 분류와 선택
- 성장기록집 주문 생성 및 상태 관리
- 주문 1건에 필요한 기록, 챕터, 메타데이터 JSON 익스포트

제외:

- 실제 인증
- 실제 인쇄 API 호출
- 결제
- 배송
- 자동 평가 또는 채용 판단
- 의료, 상담, 법률 조언
- 인쇄용 PDF 생성

## 12. 성공 지표

- 최종 승인 전 Product Fit 점수 3점 이상.
- 최종 승인 후 Product Fit 점수 4점 이상.
- Lv1/Lv2/Lv3가 하나의 사용자 여정으로 설명된다.
- 익스포트 데이터가 가상의 스위트북 파트너에게 넘길 수 있는 주문 패키지로 설명된다.
- AI 역할이 “생성자가 아니라 보조 정리자”로 제한된다.

## 13. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 성장 포트폴리오가 책으로 이어지는 이유가 약해질 수 있다. | 결과물을 “면접 자료”가 아니라 “성장 과정을 눈으로 보는 성장기록집”으로 정의한다. |
| 서비스가 자기소개서 생성기처럼 보일 수 있다. | AI는 챕터 구성, 요약, 누락 질문 제안까지만 맡기고 최종 내용은 사용자가 결정한다. |
| 타겟이 넓어져 흐름이 흐려질 수 있다. | 공통 문제를 “흩어진 성장 기록을 다시 볼 수 없음”으로 고정한다. |
| 구현 범위가 커질 수 있다. | Lv1은 성장 기록 CRUD, Lv2는 기록 선택 후 주문, Lv3는 JSON 익스포트로 제한한다. |

## 14. 권장안 요약

현재 추천 후보는 성장 포트폴리오 기반 “성장기록집” 서비스다.

이 후보는 실생활에서 필요한 기록 정리 문제를 다루면서, Book Print API의 책 전환 구조와도 연결된다. 또한 면접에서 문제 정의, 구현 범위 조절, AI 보조 활용, Lv3 데이터 구조를 설명하기 좋다.

단, 아직 최종 확정하지 않는다. 사용자가 명시적으로 승인하기 전까지 이 후보를 제품 명세, API 계약, 데이터 모델, 앱 구현에 도입하지 않는다.
