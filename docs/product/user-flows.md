# GrowthBook 사용자 플로우

## 1. 첫 실행

1. 사용자가 Docker로 앱을 실행한다.
2. 브라우저에서 `http://localhost:<port>`로 접속한다.
3. 로그인 없이 더미 성장기록과 최근 주문 상태를 본다.
4. 사용자는 GrowthBook이 “흩어진 성장 기록을 성장기록집으로 묶는 서비스”임을 첫 화면에서 이해한다.

## 2. Lv1 성장기록 플로우

1. 사용자가 성장기록 목록을 본다.
2. 카테고리와 제목을 보고 관심 있는 기록을 선택한다.
3. 상세 화면에서 문제, 행동, 배운 점, 결과, 다음 행동을 확인한다.
4. 새 성장기록을 작성한다.
5. 기존 성장기록을 수정한다.
6. 필요 없는 성장기록을 삭제한다.

완료 기준:

- 생성, 조회, 수정, 삭제가 모두 동작한다.
- 더미 데이터만으로도 서비스의 목적이 이해된다.

## 3. AI 보조 정리 플로우

1. 사용자가 성장기록 여러 개를 선택한다.
2. “성장기록집 구성 제안”을 요청한다.
3. 시스템은 챕터 제목, 챕터 요약, 포함 기록, 누락 질문을 제안한다.
4. Gemini API 키가 없으면 규칙 기반 Mock 제안을 사용한다.
5. 사용자는 제안된 챕터를 주문 생성 전에 수정할 수 있다.

완료 기준:

- API 키가 없어도 제안 기능이 동작한다.
- AI 제안은 사용자의 최종 판단을 대체하지 않는다.

## 4. Lv2 성장기록집 주문 플로우

1. 사용자가 성장기록 여러 개를 선택한다.
2. 자동 제안된 챕터 구성을 확인하고 수정한다.
3. 책 제목, 작성자명, 요청사항을 입력한다.
4. 성장기록집 주문을 생성한다.
5. 주문 목록에서 생성된 주문을 확인한다.
6. 주문 상세에서 현재 상태에 맞는 다음 행동만 선택한다.

상태 흐름:

```text
pending -> processing -> completed
pending -> cancelled
processing -> cancelled
```

완료 기준:

- 주문 생성, 목록, 상세, 상태 변경이 동작한다.
- 사용자는 종료 상태인 `completed`, `cancelled`에서 다른 상태로 되돌릴 수 없다.
- API는 UI를 우회한 잘못된 상태 변경 요청도 거부한다.

## 5. Lv3 주문 데이터 익스포트 플로우

1. 사용자가 주문 상세 화면으로 이동한다.
2. “JSON 다운로드”를 누른다.
3. 브라우저가 주문 데이터 파일을 다운로드한다.
4. JSON 안에는 주문 정보, 챕터 구성, 포함 성장기록, 익스포트 메타데이터가 들어 있다.

완료 기준:

- 주문 1건에 필요한 데이터를 한 번에 구조화해서 뽑을 수 있다.
- 익스포트 데이터는 가상의 스위트북 파트너에게 넘길 수 있는 주문 패키지처럼 읽힌다.
