# 2단계 문제 정의 실행 기록

## 1. 목적

2단계의 목적은 스위트북 과제에 사용할 콘텐츠 서비스 아이디어를 바로 확정하지 않고, 여러 실생활 문제 후보를 비교해 최종 추천 후보 1개를 도출하는 것이다.

현재 추천 후보는 성장 포트폴리오 기반 “성장기록집” 서비스다. 단, 아직 최종 승인 전이다.

## 2. 사용자 선택 내역

이번 단계에서 사용자가 명시한 방향은 아래와 같다.

- 예시에 얽매이지 않고, 실생활에서 정말 필요한 서비스였으면 좋겠다.
- 후보 평가에서는 면접 차별화를 우선한다.
- 문제의 무게감은 현실적이되 절제된 톤으로 가져간다.
- AI는 콘텐츠 생성자가 아니라 보조 정리자로 둔다.
- 대상자는 취준생, 주니어 직장인, 학생처럼 자신의 성장을 눈으로 보고 싶은 사람까지 포함한다.
- 핵심 문제는 기록이 흩어지는 것이다.
- 책의 의미는 성장기록집이다.

## 3. 비교한 후보

| 후보 | 비교 이유 | 판단 |
|---|---|---|
| 가족 돌봄 기록 | 실생활 필요성이 강하고 문제 정의가 선명하다. | 의료 조언 서비스로 오해될 수 있어 보류 |
| 가족 기억 아카이브 | 책 전환과 감성적 완성도가 좋다. | 과제 예시와 가까워 차별화가 약해질 수 있어 보류 |
| 성장 포트폴리오 | 흩어진 기록 문제, 면접 차별화, AI 보조 정리자 역할이 자연스럽다. | 추천 후보 |
| 반려동물 기록 | 실생활성과 감성이 좋고 더미 데이터가 쉽다. | 흔한 SNS/앨범 서비스로 흐를 위험이 있어 보류 |
| 생활 회고형 기록 | 구현 난이도가 낮고 범용성이 높다. | 문제 강도와 면접 차별화가 약할 수 있어 보류 |

## 4. 추천 후보

추천 후보는 성장 포트폴리오 기반 “성장기록집” 서비스다.

핵심 문제:

- 학습, 프로젝트, 회고, 실패, 개선 기록이 GitHub, Notion, 메모, 블로그, 캡처 등에 흩어져 있다.
- 사용자는 분명히 성장하고 있지만, 그 변화를 시간순으로 보거나 설명 가능한 결과물로 만들기 어렵다.

사용자:

- 취준생
- 주니어 직장인
- 대학생/학습자
- 공통점: 자신의 성장을 눈으로 확인하고, 필요할 때 포트폴리오나 회고 자료로 꺼내고 싶은 사람

서비스 방향:

- 사용자가 성장 기록을 작성하거나 모은다.
- 기록을 프로젝트, 학습, 실패, 개선, 결과 같은 축으로 정리한다.
- AI는 기록을 대신 꾸며내지 않고, 챕터 구성, 요약, 누락 질문을 제안하는 보조 정리자 역할만 한다.
- 사용자는 선택한 기록을 “성장기록집”으로 주문한다.
- Lv3에서는 성장기록집 주문 1건에 필요한 기록, 챕터, 메타데이터를 JSON으로 익스포트한다.

## 5. 최종 확정 전 금지사항

사용자가 이 후보를 명시적으로 최종 승인하기 전까지 아래 작업은 하지 않는다.

- 제품 명세에 성장기록집 아이디어를 확정 반영
- API 계약 확정
- 데이터 모델 확정
- 앱 구현 시작
- 더미 데이터 생성
- README에 최종 서비스명 또는 확정 서비스로 기재
